## Context

当前项目是一个 AI 聊天笔记应用，后端使用 Go + Gin，前端使用 Next.js。
- 对话创建时 title 字段默认为"新对话"
- 已有完整的消息发送和流式响应机制
- 已有 AI 模型调用能力（用于聊天回复）

## Goals / Non-Goals

**Goals:**
- 用户发送第一条消息后，自动生成有意义的聊天标题
- 标题生成过程不阻塞聊天流程
- 标题简洁（建议 10-20 字以内）
- 复用现有的 AI 调用能力

**Non-Goals:**
- 不支持用户手动触发生成（后续可扩展）
- 不支持多语言标题（跟随用户消息语言）
- 不做标题历史记录

## Decisions

### 1. 标题生成时机

**决定：在用户发送第一条消息后异步生成**

备选方案：
- ❌ 创建对话时同步生成：用户还没发消息，无法生成
- ❌ 每条消息后都生成：成本高，体验差
- ✅ 首条消息后异步生成：不阻塞聊天，成本可控

### 2. API 设计

**决定：新增独立 API `POST /api/conversations/:id/generate-title`**

备选方案：
- ❌ 在发送消息 API 中内置：耦合度高，流式响应复杂
- ✅ 独立 API：职责清晰，前端可灵活控制调用时机

### 3. AI 模型选择

**决定：复用当前对话使用的模型**

理由：
- 用户已选择模型，保持一致性
- 标题生成是简单任务，不需要专门模型
- 简化实现，无需额外配置

### 4. 标题生成 Prompt

```
请根据用户的第一条消息，生成一个简洁的聊天标题（最多15个字符）。
只输出标题，不要其他内容。

用户消息：{first_message}
```

## Risks / Trade-offs

**[风险] 异步生成可能延迟显示**
→ 前端在生成完成后自动刷新对话列表，延迟通常 < 2s，可接受

**[风险] AI 可能生成不合适的标题**
→ 限制标题长度，prompt 中强调简洁；用户可手动修改

**[风险] 额外的 API 调用成本**
→ 仅首条消息触发一次，成本可控
