## ADDED Requirements

### Requirement: 模型选择对话框启用/禁用功能
模型选择对话框 SHALL 为每个模型提供启用/禁用开关。

#### Scenario: 切换模型启用状态
- **WHEN** 用户在模型选择对话框中点击模型的启用开关
- **THEN** 界面显示该模型的待启用/待禁用状态
- **AND** 状态变更在点击保存前保持在本地

#### Scenario: 批量保存状态变更
- **WHEN** 用户在模型选择对话框中修改多个模型状态后点击保存
- **THEN** 系统调用 Sync API 传递所有 `enable` 和 `disable` 变更
- **AND** 保存成功后界面刷新显示最新状态

### Requirement: 模型选择对话框删除功能
模型选择对话框 SHALL 为每个模型提供删除按钮，执行硬删除。

#### Scenario: 标记模型待删除
- **WHEN** 用户在模型选择对话框中点击模型的删除按钮
- **THEN** 界面将该模型标记为待删除状态
- **AND** 模型在列表中显示删除线或灰色样式

#### Scenario: 保存时执行硬删除
- **WHEN** 用户在模型选择对话框中标记模型删除后点击保存
- **THEN** 系统调用 Sync API 传递 `delete` 数组
- **AND** 后端执行硬删除操作

## MODIFIED Requirements

### Requirement: 模型选择对话框统一管理
模型选择对话框 SHALL 作为模型管理的唯一入口，支持添加、启用/禁用、设置默认、探测、删除等所有操作。

#### Scenario: 从模型管理页面打开模型选择对话框
- **WHEN** 用户在模型管理页面点击 Provider 卡片的"管理模型"按钮
- **THEN** 系统打开模型选择对话框
- **AND** 对话框显示该 Provider 的所有模型及其当前状态

#### Scenario: 模型选择对话框显示完整功能
- **WHEN** 模型选择对话框打开
- **THEN** 每个模型行显示：模型名称、测试按钮、启用开关、设置默认按钮、删除按钮
- **AND** 管理员可在此完成所有模型管理操作
