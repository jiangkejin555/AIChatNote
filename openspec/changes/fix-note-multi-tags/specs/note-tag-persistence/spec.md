## ADDED Requirements

### Requirement: 多标签原子保存

当用户创建或更新笔记时，系统必须确保笔记内容及其所有标签作为一个原子操作进行保存。如果标签保存失败，整个操作必须回滚。

#### Scenario: 创建笔记时成功保存多个标签
- **WHEN** 用户创建一个包含多个标签 `["工作", "重要", "待办"]` 的笔记
- **THEN** 系统必须保存笔记和所有三个标签，返回 201 Created
- **AND** 数据库中该笔记必须关联所有三个标签

#### Scenario: 创建笔记时标签保存失败
- **WHEN** 用户创建笔记时标签保存因数据库错误失败
- **THEN** 系统必须回滚整个操作，笔记也不应被创建
- **AND** 返回 500 Internal Server Error
- **AND** 错误消息必须明确指出是标签保存失败

### Requirement: 标签保存错误日志

当标签保存操作失败时，系统必须记录详细的错误日志，包括笔记 ID、标签列表和错误原因。

#### Scenario: 标签保存失败时记录日志
- **WHEN** 标签保存操作失败
- **THEN** 系统必须记录包含以下信息的日志：
  - 笔记 ID
  - 尝试保存的标签列表
  - 具体的错误信息
- **AND** 日志级别为 ERROR

### Requirement: 更新笔记时标签替换

当用户更新笔记的标签时，系统必须原子性地替换所有标签。

#### Scenario: 更新笔记时成功替换标签
- **WHEN** 用户更新笔记，将标签从 `["旧标签"]` 改为 `["新标签1", "新标签2"]`
- **THEN** 系统必须删除旧标签并创建新标签
- **AND** 操作必须是原子的，不能出现部分更新
- **AND** 返回 200 OK

#### Scenario: 更新笔记时标签保存失败
- **WHEN** 用户更新笔记时新标签保存失败
- **THEN** 系统必须保持原有标签不变
- **AND** 返回 500 Internal Server Error

### Requirement: 事务隔离保证

笔记创建和标签创建必须在同一个数据库事务中执行，确保数据一致性。

#### Scenario: 事务回滚验证
- **WHEN** 在保存标签之前发生任何错误
- **THEN** 事务必须完全回滚
- **AND** 数据库中不能存在孤立的笔记记录
