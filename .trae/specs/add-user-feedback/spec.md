# 关于我们页面重构 Spec

## Why

当前"关于我们"页面将帮助、反馈、版本三个功能放在一起，功能定位不够清晰。需要重构为：
1. "关于我们"专注于产品介绍、公司信息、团队介绍等静态信息
2. "帮助与反馈"作为独立入口，包含帮助文档和用户反馈功能

## What Changes

- **重构"关于我们"页面**：新增产品介绍、公司信息、团队介绍、联系方式、服务条款/隐私政策
- **新增"帮助与反馈"入口**：合并帮助和反馈功能
- **更新侧边栏菜单**：添加"帮助与反馈"入口，调整"关于我们"位置
- **版本信息**：移至"关于我们"页面底部

## Impact

- Affected specs: 无
- Affected code:
  - 前端: 重构 `frontend/src/app/(main)/about/` 页面结构
  - 前端: 新增 `frontend/src/app/(main)/help/` 页面
  - 前端: `frontend/src/components/layout/sidebar.tsx` 更新菜单
  - 国际化: `frontend/messages/zh.json`, `frontend/messages/en.json`

## ADDED Requirements

### Requirement: 关于我们页面重构

系统应提供"关于我们"页面，展示产品和公司的静态信息。

#### Scenario: 用户访问关于我们页面
- **WHEN** 用户点击侧边栏用户菜单中的"关于我们"
- **THEN** 系统跳转到 `/about` 页面
- **AND** 页面包含以下内容：
  - 产品介绍
  - 公司信息
  - 团队介绍
  - 联系方式
  - 服务条款/隐私政策链接
  - 版本信息

### Requirement: 帮助与反馈页面

系统应提供"帮助与反馈"页面，整合帮助文档和用户反馈功能。

#### Scenario: 用户访问帮助与反馈页面
- **WHEN** 用户点击侧边栏用户菜单中的"帮助与反馈"
- **THEN** 系统跳转到 `/help` 页面
- **AND** 页面顶部显示两个 Tab：帮助、反馈
- **AND** 默认显示帮助内容

#### Scenario: 用户切换帮助和反馈 Tab
- **WHEN** 用户点击"反馈"Tab
- **THEN** 系统显示反馈内容（满意度评分、问题反馈、功能投票）

### Requirement: 侧边栏菜单更新

侧边栏用户菜单应包含新的入口结构。

#### Scenario: 用户查看侧边栏菜单
- **WHEN** 用户点击侧边栏用户头像
- **THEN** 系统显示以下菜单项：
  - 设置
  - 帮助与反馈 ← 新增
  - ───────────
  - 关于我们
  - ───────────
  - 退出登录

需要在翻译文件中添加以下新键：

```json
{
  "about": {
    "productIntro": "产品介绍",
    "productIntroContent": "...",
    "companyInfo": "公司信息",
    "companyName": "公司名称",
    "foundedDate": "成立时间",
    "companyMission": "公司使命",
    "teamIntro": "团队介绍",
    "teamIntroContent": "...",
    "contact": "联系方式",
    "email": "邮箱",
    "legalInfo": "法律信息",
    "termsOfService": "服务条款",
    "privacyPolicy": "隐私政策",
    "versionInfo": "版本信息"
  },
  "helpFeedback": {
    "title": "帮助与反馈",
    "helpTab": "帮助",
    "feedbackTab": "反馈"
  }
}
```
