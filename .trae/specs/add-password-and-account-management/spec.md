# 修改密码和注销账号功能 Spec

## Why

当前系统缺少账号管理功能，用户无法修改密码或注销账号。这导致：
1. 用户无法更改密码，存在安全隐患
2. 用户无法删除账号和数据，不符合数据保护法规要求
3. 缺少账号管理入口，用户体验不完整

通过实现修改密码和注销账号功能，可以：
1. 提升账号安全性（用户可以定期更换密码）
2. 符合数据保护法规（GDPR 等）
3. 完善用户体验

## What Changes

- **新增修改密码功能**：用户通过验证码验证后修改密码
- **新增注销账号功能**：用户可以删除账号和所有数据
- **新增账号管理入口**：在头像下拉菜单中添加"账号管理"选项
- **新增前端对话框**：修改密码和注销账号确认对话框

## Impact

- Affected specs: add-oauth-login, add-email-verification-login
- Affected code:
  - 后端: `backend/internal/handlers/auth.go` 新增修改密码和注销账号接口
  - 后端: `backend/internal/repository/user.go` 新增密码更新和用户删除方法
  - 前端: `frontend/src/components/layout/sidebar.tsx` 添加"账号管理"菜单项
  - 前端: `frontend/src/components/auth/` 添加修改密码和注销账号对话框
  - 前端: `frontend/src/lib/api/auth.ts` 新增 API 方法

## ADDED Requirements

### Requirement: 修改密码功能

系统应支持用户通过验证码验证后修改登录密码。

#### Scenario: 用户请求修改密码
- **WHEN** 用户在账号管理中点击"修改密码"
- **THEN** 系统显示修改密码表单
- **AND** 要求输入邮箱验证码和新密码

#### Scenario: 用户发送验证码
- **WHEN** 用户点击"发送验证码"
- **THEN** 系统发送6位验证码到用户邮箱
- **AND** 显示60秒倒计时

#### Scenario: 用户提交修改密码
- **WHEN** 用户提交修改密码表单
- **THEN** 系统验证验证码是否正确
- **AND** 验证新密码长度（至少8位）
- **AND** 更新用户密码
- **AND** 返回成功消息

#### Scenario: 验证码错误
- **WHEN** 用户输入的验证码错误或过期
- **THEN** 系统返回错误提示"验证码错误或已过期"
- **AND** 不修改密码

#### Scenario: OAuth 用户修改密码
- **WHEN** OAuth 登录用户尝试修改密码
- **THEN** 系统提示"OAuth 登录用户无密码，请使用 OAuth 登录"
- **AND** 不显示修改密码选项

### Requirement: 注销账号功能

系统应支持用户注销账号并删除所有数据。

#### Scenario: 用户请求注销账号
- **WHEN** 用户在账号管理中点击"注销账号"
- **THEN** 系统显示注销确认对话框
- **AND** 提示"此操作将永久删除您的账户和所有数据，且不可恢复"

#### Scenario: 用户确认注销账号
- **WHEN** 用户确认注销账号
- **THEN** 系统要求用户输入密码或进行 OAuth 验证
- **AND** 验证通过后删除用户所有数据
- **AND** 清除用户登录状态
- **AND** 返回成功消息
- **AND** 前端跳转到登录页面

#### Scenario: 注销账号数据删除
- **WHEN** 用户确认注销账号
- **THEN** 系统删除以下数据：
  - 用户的所有会话（conversations）
  - 用户的所有消息（messages）
  - 用户的所有笔记（notes）
  - 用户的所有文件夹（folders）
  - 用户的所有标签（note_tags）
  - 用户的所有模型配置（providers, provider_models）
  - 用户的所有 OAuth 关联（oauth_accounts）
  - 用户的所有刷新令牌（refresh_tokens）
  - 用户的所有消息请求记录（message_requests）
  - 用户的所有会话摘要（conversation_summaries）
  - 用户的所有用户设置（user_settings）
  - 用户的所有反馈数据（feedbacks, satisfaction_ratings, feature_requests）
  - 用户账号本身（users）

#### Scenario: 注销账号失败
- **WHEN** 注销过程中发生错误
- **THEN** 系统回滚所有已删除的数据
- **AND** 系统记录错误日志
- **AND** 提示用户"注销失败，请稍后重试"

### Requirement: 头像下拉菜单账号管理入口

前端应在头像下拉菜单中提供账号管理入口。

#### Scenario: 用户点击头像
- **WHEN** 用户点击头像

- **AND** 包含"账号管理"选项

- **AND** 包含"关于"选项
- **AND** 不包含"退出登录"选项（退出登录在账号管理对话框中）

#### Scenario: 用户点击账号管理
- **WHEN** 用户点击"账号管理"
- **THEN** 显示账号管理对话框
- **AND** 显示"修改密码"按钮（仅邮箱登录用户）
- **AND** 显示"退出登录"按钮
- **AND** 显示"注销账号"按钮（在"退出登录"下方）

#### Scenario: 修改密码表单
- **WHEN** 用户点击"修改密码"
- **THEN** 显示修改密码表单
- **AND** 显示用户邮箱（只读）

#### Scenario: 注销账号确认对话框

- **THEN** 显示确认对话框

## API 设计

### 新增接口

#### 1. 修改密码（使用验证码）
```
PUT /api/auth/password
Request: {
  "email": "user@example.com",
  "code": "123456",
  "new_password": "newpassword456"
}
Response: {
  "message": "Password updated successfully"
}
```

**注意**：
- 验证码通过现有的 `/api/auth/email/code` 接口发送
- 新密码长度至少8位

#### 2. 注销账号
```
DELETE /api/auth/account
Request: {
  "password": "userpassword123" // 可选，邮箱登录用户需要
}
Response: {
  "message": "Account deleted successfully"
}
```

**注意**：
- 邮箱登录用户需要提供密码验证
- OAuth 登录用户需要重新进行 OAuth 验证

## 安全性考虑

1. **修改密码安全**
   - 验证码验证（6位，10分钟过期）
   - 新密码长度验证（至少8位）
   - 密码加密存储（bcrypt）

2. **注销账号安全**
   - 密码验证或 OAuth 验证
   - 事务处理确保数据一致性
   - 删除后无法恢复

## 风险与缓解

### 风险 1: 用户误删账号
- **影响**: 用户数据永久丢失
- **缓解**: 显示明确的警告信息，要求二次确认

### 风险 2: 注销过程中断
- **影响**: 数据部分删除
- **缓解**: 使用事务处理，失败时回滚

### 风险 3: 恶意注销他人账号
- **影响**: 用户账号被盗用后删除
- **缓解**: 要求密码验证或 OAuth 验证

## 回滚计划

如需回滚功能：
1. 移除相关路由
2. 隐藏前端按钮
3. 用户仍可正常使用其他功能
