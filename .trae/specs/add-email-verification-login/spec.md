# 邮箱验证码登录功能 Spec

## Why

当前系统仅支持邮箱+密码登录和 OAuth 登录。邮箱+密码登录存在以下问题：
1. 邮箱没有实际验证，用户可以随意填写不存在的邮箱
2. 用户需要记住额外的密码，体验不佳
3. 没有密码找回功能

通过实现邮箱验证码登录，可以：
1. 确保邮箱真实有效（必须能收到验证码）
2. 无需密码，提升用户体验
3. 更安全（验证码有时效性）
4. 作为 OAuth 登录的补充方案

## What Changes

- **新增邮箱验证码发送功能**：用户输入邮箱后发送 6 位验证码
- **新增邮箱验证码登录功能**：使用验证码登录/注册
- **保留邮箱+密码登录**：作为备选方案
- **新增 SMTP 邮件服务**：使用 Gmail SMTP 发送邮件
- **新增验证码管理**：生成、存储、验证验证码

## Impact

- Affected specs: add-oauth-login
- Affected code:
  - 后端: `backend/internal/config/` 新增 SMTP 配置
  - 后端: `backend/internal/services/` 新增邮件服务和验证码服务
  - 后端: `backend/internal/handlers/auth.go` 新增验证码相关接口
  - 前端: `frontend/src/app/(auth)/login/` 添加验证码登录表单
  - 前端: `frontend/src/lib/api/auth.ts` 新增验证码 API
  - 配置: `backend/config.yaml` 新增 SMTP 配置

## ADDED Requirements

### Requirement: SMTP 邮件服务配置

系统应支持 SMTP 邮件服务配置。

#### Scenario: 管理员配置 SMTP 服务
- **WHEN** 管理员在 config.yaml 中配置 SMTP 信息
- **THEN** 系统应加载并验证配置
- **AND** 配置包括：SMTP 服务器地址、端口、用户名、密码、发件人地址

#### Scenario: SMTP 配置验证
- **WHEN** 系统启动时
- **THEN** 系统应验证 SMTP 配置的完整性
- **AND** 对于缺失的配置，应记录警告日志

### Requirement: 邮箱验证码生成与发送

系统应能够生成并发送邮箱验证码。

#### Scenario: 用户请求验证码
- **WHEN** 用户输入邮箱并点击"发送验证码"
- **THEN** 系统验证邮箱格式
- **AND** 生成 6 位数字验证码
- **AND** 存储验证码（有效期 10 分钟）
- **AND** 发送验证码到用户邮箱
- **AND** 返回成功消息

#### Scenario: 验证码生成规则
- **WHEN** 系统生成验证码
- **THEN** 验证码应为 6 位数字
- **AND** 验证码应随机生成
- **AND** 验证码应与邮箱绑定存储

#### Scenario: 验证码发送失败
- **WHEN** 邮件发送失败
- **THEN** 系统记录错误日志
- **AND** 返回错误提示"验证码发送失败，请稍后重试"

### Requirement: 邮箱验证码登录

系统应支持使用邮箱验证码登录。

#### Scenario: 用户使用验证码登录
- **WHEN** 用户输入邮箱和验证码
- **THEN** 系统验证验证码有效性
- **AND** 验证码未过期（10 分钟内）
- **AND** 验证码正确
- **AND** 检查用户是否存在
- **AND** 如用户不存在，创建新用户
- **AND** 生成 JWT token
- **AND** 删除已使用的验证码
- **AND** 返回登录成功

#### Scenario: 验证码错误
- **WHEN** 用户输入的验证码错误
- **THEN** 系统返回错误提示"验证码错误"
- **AND** 不删除验证码（允许重试）

#### Scenario: 验证码过期
- **WHEN** 用户输入的验证码已过期
- **THEN** 系统返回错误提示"验证码已过期，请重新获取"
- **AND** 删除过期的验证码

### Requirement: 验证码管理

系统应管理验证码的生命周期。

#### Scenario: 验证码存储
- **WHEN** 系统生成验证码
- **THEN** 验证码应存储在内存中
- **AND** 键格式：`email_code:{email}`
- **AND** 值为验证码
- **AND** 设置过期时间 10 分钟

#### Scenario: 验证码清理
- **WHEN** 验证码使用成功或过期
- **THEN** 系统应删除验证码
- **AND** 防止重复使用

#### Scenario: 验证码重发限制
- **WHEN** 用户在 60 秒内重复请求验证码
- **THEN** 系统拒绝发送
- **AND** 返回提示"验证码已发送，请稍后再试"

### Requirement: 前端验证码登录流程

前端应支持完整的验证码登录流程。

#### Scenario: 用户使用验证码登录
- **WHEN** 用户在登录页面选择"验证码登录"
- **THEN** 显示邮箱输入框和验证码输入框
- **AND** 显示"发送验证码"按钮
- **AND** 点击按钮后调用发送验证码 API
- **AND** 显示 60 秒倒计时
- **AND** 输入验证码后点击登录
- **AND** 调用验证码登录 API
- **AND** 登录成功后跳转

#### Scenario: 验证码登录失败
- **WHEN** 验证码登录失败
- **THEN** 显示错误提示
- **AND** 提供重新发送验证码选项

## MODIFIED Requirements

### Requirement: 登录页面改造

登录页面应同时支持密码登录和验证码登录。

#### Scenario: 用户访问登录页面
- **WHEN** 用户访问 `/login` 页面
- **THEN** 页面显示登录方式切换 Tab
  - 密码登录
  - 验证码登录
- **AND** 默认显示密码登录

### Requirement: 用户模型调整

用户模型应支持验证码登录。

#### Scenario: 用户信息
- **WHEN** 用户通过验证码登录
- **THEN** 用户模型应包含：
  - `email`: 用户邮箱（已验证）
  - `email_verified`: 设置为 true
  - `nickname`: 可选，用户可后续设置
  - `password_hash`: 可为空（验证码登录用户无密码）

## 配置设计

### config.yaml 新增配置

```yaml
smtp:
  host: smtp.gmail.com
  port: 587
  username: your-email@gmail.com
  password: your-app-password
  from_email: your-email@gmail.com
  from_name: AI Chat Note
```

### 环境变量支持

```bash
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USERNAME=your-email@gmail.com
SMTP_PASSWORD=your-app-password
SMTP_FROM_EMAIL=your-email@gmail.com
SMTP_FROM_NAME=AI Chat Note
```

## API 设计

### 新增接口

#### 1. 发送验证码
```
POST /api/auth/email/code
Request: {
  "email": "user@example.com"
}
Response: {
  "message": "验证码已发送"
}
```

#### 2. 验证码登录
```
POST /api/auth/email/login
Request: {
  "email": "user@example.com",
  "code": "123456"
}
Response: {
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "refresh_token": "def456...",
  "user": {...}
}
```

## 安全性考虑

1. **验证码安全**
   - 6 位数字，随机生成
   - 有效期 10 分钟
   - 使用后立即删除
   - 最多重试 5 次

2. **发送限制**
   - 同一邮箱 60 秒内只能发送一次
   - 同一 IP 每小时最多发送 10 次

3. **邮箱验证**
   - 验证邮箱格式
   - 记录邮箱验证状态

## 风险与缓解

### 风险 1: 邮件发送失败
- **影响**: 用户无法收到验证码
- **缓解**: 提供重试选项，记录失败日志

### 风险 2: 验证码被拦截
- **影响**: 用户收不到验证码
- **缓解**: 提示用户检查垃圾邮件，提供重发选项

### 风险 3: SMTP 服务不可用
- **影响**: 无法发送邮件
- **缓解**: 保留密码登录作为备选方案

## 回滚计划

如需回滚验证码登录功能：
1. 移除验证码相关路由
2. 隐藏前端验证码登录选项
3. 保留 SMTP 配置（不影响现有功能）
4. 用户仍可使用密码登录和 OAuth 登录
